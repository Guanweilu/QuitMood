# QuitMood 种草视频剪辑 · 同事使用指南

> 这份文档是给完全没用过 AI 剪辑工具的同事看的。
> 不需要懂代码，按步骤来就行。看不懂的地方截图问 Codex。

---

## 你会得到什么？

把你拍的产品素材丢进去，AI 自动帮你剪成 **15 秒的种草短视频**，竖屏，带配音，可以直接发小红书/抖音。

---

## 第一步：安装 FFmpeg（只需要做一次）

FFmpeg 是剪视频用的底层工具，必须装。

### Mac 用户

打开你电脑的 **终端**（在启动台搜"终端"或"Terminal"）。

**如果你有管理员权限**（大部分人都有），把下面这段复制粘贴进终端，按回车：

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

装完后再跑：

```bash
brew install ffmpeg
```

**如果提示没有权限**，用这个方式（不需要管理员）：

```bash
mkdir -p ~/bin
curl -L -o ~/ffmpeg.zip https://evermeet.cx/ffmpeg/ffmpeg-7.1.1.zip
unzip -o ~/ffmpeg.zip -d ~/bin/
chmod +x ~/bin/ffmpeg
rm ~/ffmpeg.zip
```

### 怎么确认装好了？

终端里输入：

```bash
ffmpeg -version
```

或者（如果你用的是无管理员的方式）：

```bash
~/bin/ffmpeg -version
```

看到版本号就说明装好了。**如果报错，截图发给 Codex，让它帮你看。**

---

## 第二步：下载剪辑工具

在终端里跑：

```bash
git clone https://github.com/Guanweilu/QuitMood.git
```

这会在你电脑上创建一个 `QuitMood` 文件夹，里面有剪辑脚本。

### 怎么确认下载好了？

```bash
ls QuitMood/
```

应该能看到 `种草剪辑.py`、`SKILL.md`、`README.md` 这几个文件。

---

## 第三步：把 Skill 文件喂给 Codex

这是最关键的一步。Skill 文件告诉 AI 怎么剪视频、遵守什么规则。

### 操作方法

1. 打开 Codex
2. 把下面这段话**完整复制粘贴**给 Codex：

```
请你阅读并严格遵守以下 Skill 文件中的所有规则。这是一个视频剪辑工具的操作规范，后续我让你帮我剪视频时，你必须按照这个规范来执行，不能自己发挥。

Skill 文件地址：https://github.com/Guanweilu/QuitMood/blob/main/SKILL.md

请你现在就读一遍，读完后告诉我你理解了哪些关键规则。
```

3. Codex 会回复它读到的规则。**你检查一下它有没有提到这几条**（如果没提到，让它重新读）：
   - ✅ 每个视频必须走完产品的完整使用流程
   - ✅ 多个视频之间只换角度，不换步骤
   - ✅ 重复镜头只取最后一个
   - ✅ 成品中绝不出现重复镜头
   - ✅ 叙事逻辑必须通顺
   - ✅ 开头不要太碎

**如果它都提到了，说明它理解了，可以开始干活。**

---

## 第四步：准备素材

### 你需要准备什么

1. **素材视频**：你拍的产品视频（.MOV 或 .mp4），放在一个文件夹里
2. **配音音频**：（可选）一段 .MP3 配音

### 素材怎么放

把所有素材视频放在同一个文件夹里就行，不需要改文件名。

---

## 第五步：让 Codex 帮你剪

把下面这段话发给 Codex（把路径换成你自己的）：

```
请按照 SKILL.md 的规则，帮我剪辑种草视频。

素材文件夹：/Users/你的用户名/Downloads/你的素材文件夹名/
音频文件：/Users/你的用户名/Downloads/你的音频.MP3
产品类型：功能型（B）
需要几个成品：2

请严格按照 Skill 中的规则来执行：
1. 先用 FFmpeg 的场景检测分析素材
2. 重复镜头只取最后一个
3. 每个视频都要走完产品的完整使用流程
4. 多个视频只换角度不换步骤
5. 成品中不能有重复镜头

剪辑脚本在：~/QuitMood/种草剪辑.py
FFmpeg 路径：~/bin/ffmpeg（如果这个路径不存在就用 ffmpeg）
输出到：~/QuitMood/输出/
```

然后等它执行完就好。

---

## 第六步：检查成品（Double Check）

剪完后，发给 Codex 这段话确认：

```
请帮我检查一下刚才剪出来的视频：

1. 一共生成了几个成品？
2. 每个成品的时长是多少秒？
3. 每个成品是否都包含了完整的产品使用流程？（从头到尾的步骤都有）
4. 不同成品之间有没有使用重复的镜头？
5. 有没有音频？音频和视频是否同步？
6. 输出的视频分辨率是多少？是不是竖屏 1080x1920？

请逐条回答。
```

如果 6 条都通过，就可以拿去用了。

## 常见问题

### Q: 终端在哪里？
Mac 上：打开 **启动台**（Launchpad）→ 搜索 **终端** 或 **Terminal** → 打开它。

### Q: 终端里的命令跑完报错了怎么办？
把报错信息截图，发给 Codex，问它："这个报错是什么意思，怎么解决？"

### Q: 不知道自己的文件路径怎么写？
把文件从 Finder 直接**拖到终端窗口**里，它会自动显示完整路径。

### Q: 想换横屏怎么办？
告诉 Codex："我想要横屏 1920x1080 的输出"，它会帮你改。

### Q: 跑脚本提示 python3 找不到？
终端里跑 `python3 --version`，如果报错说明没装 Python。问 Codex："我的 Mac 怎么安装 Python 3？"

### Q: 产品类型怎么选？
- **模式 A（外观型）**：产品外观是卖点的，比如润唇膏、香水、化妆品
- **模式 B（功能型）**：产品使用过程是卖点的，比如煮蛋器、牙贴、清洁工具

### Q: 素材很多，不知道拍的什么了怎么办？
先在 Finder 里双击每个视频快速预览一遍，把明显拍坏的删掉，再丢给 AI。
