# QuitMood 种草剪辑 Skill v6.0

## 核心理念

**每个种草视频必须清晰回答消费者的三个问题：**

> **1. 这个产品长啥样？**
> **2. 怎么用？**
> **3. 有什么用？（效果是什么）**

这三个问题就是叙事的全部。镜头选择、顺序、时长，一切都围绕这三个问题展开。

**不要盲剪，先看素材再动手。** 自动场景检测只能找到画面变化点，不能判断画面好不好、放哪里合适。必须先抽帧预览，理解每条素材拍了什么，再制定剪辑方案。

## 触发时的完整流程

### 第 1 步：收集信息

向用户要：
- 素材文件夹路径
- 音频路径（可选）
- 产品类型（外观型 A / 功能型 B，影响时长分配）
- 要几个成品
- 有无特殊要求（比如"搓手指的画面放最后"）

### 第 2 步：抽帧预览

对每条素材抽取缩略图，看懂每条拍了什么：

```bash
mkdir -p /tmp/qm_preview
for f in 素材文件夹/IMG_*.MOV; do
  name=$(basename "$f" .MOV)
  ~/bin/ffmpeg -y -i "$f" -vf "fps=1/2,scale=320:-1,tile=5x1" -frames:v 1 "/tmp/qm_preview/${name}.jpg" 2>/dev/null
done
```

然后用 Read 工具查看每张缩略图。

### 第 3 步：分析素材 & 制定方案

看完缩略图后，为每条素材标注它回答的是哪个问题：

| 问题 | 对应的画面类型 |
|------|--------------|
| **长啥样？** | 产品全貌、外包装、多角度展示、拆箱开封 |
| **怎么用？** | 上手操作、涂抹、安装、启动、使用过程 |
| **有什么用？** | 使用前后对比、效果特写、成品展示 |

然后制定方案时，确保每个视频都**按顺序**覆盖这三个问题。

### 叙事弧线：三段式结构

**不要锁死固定的段落模板。** 根据产品实际步骤来定镜头数和时长。总时长控制在 **10-18 秒区间**，不强制 15 秒。

**结构原则：**

```
第一段「长啥样」（占总时长 25-35%）
  → 产品第一眼 / 外包装 / 拆封 / 内部展示
  → 2-3 个镜头

第二段「怎么用」（占总时长 35-45%）
  → 使用步骤，每个镜头是一个不同的操作动作
  → 3-4 个镜头

第三段「有什么用」（占总时长 20-30%）
  → 效果展示 / 成品 / CTA
  → 1-2 个镜头
```

**模式 A · 外观型（润唇膏、香水等）：**
「长啥样」占比拉大（35%），因为外观本身就是卖点。

**模式 B · 功能型（煮蛋器、牙贴等）：**
「怎么用」占比拉大（45%），因为使用过程才是卖点。

### 第 4 步：执行剪辑

用 FFmpeg 逐段截取 + 拼接 + 合音频：

```bash
FF=~/bin/ffmpeg
W=1080; H=1920; FPS=30
VF="scale=${W}:${H}:force_original_aspect_ratio=increase,crop=${W}:${H},fps=${FPS},setsar=1"

# 逐段截取
$FF -y -ss {开始时间} -i {素材路径} -t {时长} -vf "$VF" -an -c:v libx264 -preset fast -crf 23 -pix_fmt yuv420p {输出段落}.mp4

# 拼接
$FF -y -f concat -safe 0 -i filelist.txt -c copy tmp.mp4

# 合音频
$FF -y -i tmp.mp4 -i 音频.MP3 -c:v copy -c:a aac -shortest 成品.mp4
```

### 第 5 步：交付

告诉用户成品路径，让他们查看效果。如果不满意，根据反馈调整方案重新剪。

## 重要原则（血泪教训，必须遵守）

### 原则 1：永远先看素材再剪
不要直接跑自动脚本，除非用户明确要求"快速出片不用看"。自动模式会盲选镜头，效果远不如看完素材手动制定方案。

### 原则 2：每个视频必须走完产品的完整故事
回答「长啥样→怎么用→有什么用」三个问题。**不能只回答一两个就结束。** 比如不能只有展示没有使用，也不能只有使用没有效果。

### 原则 3：多个视频之间只换角度，不换步骤
三个视频的叙事框架完全一样，区别只在于同一个步骤选了不同的拍摄角度/时间段。用户拍了很多条素材，就是为了同一个动作有多个角度可选。

### 原则 4：每个镜头必须推进故事
**8 个镜头 = 8 个不同的步骤。** 不能在同一个步骤上停留超过一个镜头。如果两个候选镜头的动作是同一件事（比如都是"手拿产品展示"），只能用一个，另一个位置必须给下一个步骤。

判断标准：如果把这个镜头去掉，故事会缺一步吗？如果不会，说明这个镜头是多余的。

### 原则 5：叙事逻辑必须通顺
不能出现"盖了盖子又打开"这种违反使用顺序的画面。每个镜头的内容必须和上下文衔接，放在这个位置要合理。

### 原则 6：时长不锁死，根据产品步骤来定
总时长控制在 **10-18 秒**。步骤多的产品可以长一点，步骤少的短一点。不要为了凑 15 秒而重复镜头或加无意义的画面。

### 原则 7：开头不要太碎
前半段不要全是 1.5 秒的短切。开头几段应该从动作的**自然起点**开始（通常是素材的 0 秒），给足呼吸。可以适当拉长到 2-2.5 秒。从素材中间位置截取容易导致动作不连贯。

### 原则 8：注意用户的特殊要求
比如"搓手指放最后"、"某个画面不要用"、"开盖画面去掉"。这些细节决定了视频的专业感。

### 原则 9：CTA 段永远放最后
引导点击的手势/画面不能出现在中间。

### 原则 10：同一条素材中有重复镜头时，只取最后一个
用户拍摄习惯是同一个动作拍多遍，一镜到底不停机。前面的都是"练习"，最后一遍才是满意的效果。所以当场景检测发现一条素材内有多个相似镜头时，**丢弃前面的，只保留最后一个**。

### 原则 11：成品中绝不出现重复镜头
严格去重。无论是同一条素材的不同时间段看起来一样，还是不同成品之间撞镜头，都不允许。如果可用镜头不够，宁可减少成品数量，也不要用重复画面凑数。

### 原则 12：镜头构图要有变化
连续的镜头不能都是同一种构图（比如连续三个都是"手拿产品正面"）。要在远景/近景/特写/使用画面之间交替，让视觉有节奏。

## 质量自检清单

制定方案后，逐条检查：

- [ ] 视频是否回答了「长啥样」？
- [ ] 视频是否回答了「怎么用」？
- [ ] 视频是否回答了「有什么用」？
- [ ] 每个镜头是否都在推进故事？（有没有同一步骤占了两个位置）
- [ ] 叙事顺序是否通顺？（有没有前后矛盾）
- [ ] 有没有重复镜头？
- [ ] 连续镜头的构图是否有变化？
- [ ] 总时长是否在 10-18 秒区间？
