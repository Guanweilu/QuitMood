# QuitMood 种草剪辑 Skill v6.0

## 核心理念

**每个种草视频必须清晰回答消费者的三个问题：**

> **1. 这个产品长啥样？**
> **2. 怎么用？**
> **3. 有什么用？（效果是什么）**

这三个问题就是叙事的全部。镜头选择、顺序、时长，一切都围绕这三个问题展开。

**不要盲剪，先看素材再动手。** 自动场景检测只能找到画面变化点，不能判断画面好不好、放哪里合适。必须先抽帧预览，理解每条素材拍了什么，再制定剪辑方案。

## 工具位置

- 脚本: `~/QuitMood/种草剪辑.py`（自动模式，适合快速出片）

## 触发时的完整流程

### 第 1 步：收集信息

向用户要：
- 素材文件夹路径
- 音频路径（可选）
- 产品类型（外观型 A / 功能型 B，影响时长分配）
- 要几个成品
- 有无特殊要求（比如"搓手指的画面放最后"）

### 第 2 步：抽帧预览

对每条素材抽取缩略图，看懂每条拍了什么：

```bash
mkdir -p /tmp/qm_preview
for f in 素材文件夹/IMG_*.MOV; do
  name=$(basename "$f" .MOV)
  ~/bin/ffmpeg -y -i "$f" -vf "fps=1/2,scale=320:-1,tile=5x1" -frames:v 1 "/tmp/qm_preview/${name}.jpg" 2>/dev/null
done
```

然后用 Read 工具查看每张缩略图。

### 第 3 步：分析素材 & 制定方案

看完缩略图后，为每条素材标注它回答的是哪个问题：

| 问题 | 对应的画面类型 |
|------|--------------|
| **长啥样？** | 产品全貌、外包装、多角度展示、拆箱开封 |
| **怎么用？** | 上手操作、涂抹、安装、启动、使用过程 |
| **有什么用？** | 使用前后对比、效果特写、成品展示 |

然后制定方案时，确保每个视频都**按顺序**覆盖这三个问题。

### 第 3.5 步（新流程）：关键词方案确认

**不要直接出剪辑表格，给用户一个关键词方案。** 用户只需要 5 秒扫一眼就能判断对不对。

格式示例：
```
1. 礼盒正面展示（手滑入）
2. 开盒看12罐排列
3. 近景手指滑过各色罐子
4. 拿起一罐近景看
5. 打开盖子看膏体
6. 手抓一把多色展示
7. 涂手背看质地
8. 手指点击收尾
```

每行一个镜头，用最短的话描述"这个镜头里在干什么"。用户说 OK 就剪，用户说要调就按反馈改。**不要给用户看时间戳、文件名、技术参数，他们不看这些。**

### 拍摄建议（提升出片质量的关键）

**素材质量决定了出片质量。** 最好的素材是：每条只包含一个干净的动作，拍完暂停再拍下一个。

好的素材结构：
```
clip_01: 礼盒正面展示
clip_02: 开盒
clip_03: 近景看细节
clip_04: 拿起单品
clip_05: 打开盖子
clip_06: 使用/涂抹
clip_07: 效果展示
clip_08: CTA手势
```

每条素材 = 一个动作 = 一个镜头。这样不需要从长素材里猜哪一段好，直接按顺序拼就流畅。

### 核心剪辑逻辑：音频卡点 > 固定时长

**画面跟着音频走，不是音频贴上去凑合。**

以前的做法（错误）：先定好每段画面的时长 → 最后把音频整条贴上去 → 画面和声音各走各的，"两层皮"。

正确做法：**先分析音频节奏，每句话对应一个镜头，画面切换踩在换句的时刻。**

### 第 3.5 步：分析音频节奏（有音频时必做）

如果用户提供了配音音频，**在制定剪辑方案之前**，先用 Whisper 识别音频：

```bash
# 在 Python 中
import whisper
model = whisper.load_model("base")
result = model.transcribe("音频.MP3", language="zh")
for seg in result["segments"]:
    print(f'{seg["start"]:.1f}s - {seg["end"]:.1f}s  {seg["text"]}')
```

得到每句话的起止时间，比如：
```
0.0s - 2.1s  "这个煮蛋器也太方便了吧"
2.1s - 4.5s  "把鸡蛋放进去加点水"
4.5s - 6.8s  "盖上盖子按一下开关"
6.8s - 8.5s  "十分钟就煮好了"
8.5s - 10.7s "剥开看看 溏心蛋完美"
```

**然后按每句话来分配镜头：**
- 句数 = 镜头数
- 每句话的时长 = 对应镜头的时长
- 总时长 = 音频时长（不用自己定 15 秒还是 18 秒）

**这样画面切换的时刻和配音换句的时刻完全重合，看起来就是一体的、流畅的。**

### 叙事弧线：三段式结构

即使镜头时长由音频决定，叙事顺序仍然要遵守三段式：

```
音频前几句 → 匹配「长啥样」的画面
音频中间句 → 匹配「怎么用」的画面
音频最后句 → 匹配「有什么用」的画面
```

**模式 A · 外观型（润唇膏、香水等）：**
「长啥样」的句子/镜头可以多一些。

**模式 B · 功能型（煮蛋器、牙贴等）：**
「怎么用」的句子/镜头可以多一些。

### 无音频时的兜底方案

如果没有配音音频，则回到手动定时长的方式：
- 总时长 10-18 秒区间
- 5-8 个镜头
- 快-慢-快-慢交替节奏

### 第 4 步：执行剪辑

用 FFmpeg 逐段截取 + 拼接 + 合音频：

```bash
FF=~/bin/ffmpeg
W=1080; H=1920; FPS=30
VF="scale=${W}:${H}:force_original_aspect_ratio=increase,crop=${W}:${H},fps=${FPS},setsar=1"

# 逐段截取（每段时长由音频句子决定）
$FF -y -ss {开始时间} -i {素材路径} -t {这句话的时长} -vf "$VF" -an -c:v libx264 -preset fast -crf 23 -pix_fmt yuv420p {输出段落}.mp4

# 拼接
$FF -y -f concat -safe 0 -i filelist.txt -c copy tmp.mp4

# 合音频
$FF -y -i tmp.mp4 -i 音频.MP3 -c:v copy -c:a aac -shortest 成品.mp4
```

### 第 5 步：交付

告诉用户成品路径，让他们查看效果。如果不满意，根据反馈调整方案重新剪。

## 重要原则（血泪教训，必须遵守）

### 原则 1：永远先看素材再剪
不要直接跑自动脚本，除非用户明确要求"快速出片不用看"。自动模式会盲选镜头，效果远不如看完素材手动制定方案。

### 原则 2：每个视频必须走完产品的完整故事
回答「长啥样→怎么用→有什么用」三个问题。**不能只回答一两个就结束。** 比如不能只有展示没有使用，也不能只有使用没有效果。

### 原则 3：多个视频之间只换角度，不换步骤
三个视频的叙事框架完全一样，区别只在于同一个步骤选了不同的拍摄角度/时间段。用户拍了很多条素材，就是为了同一个动作有多个角度可选。

### 原则 4：每个镜头必须推进故事
**8 个镜头 = 8 个不同的步骤。** 不能在同一个步骤上停留超过一个镜头。如果两个候选镜头的动作是同一件事（比如都是"手拿产品展示"），只能用一个，另一个位置必须给下一个步骤。

判断标准：如果把这个镜头去掉，故事会缺一步吗？如果不会，说明这个镜头是多余的。

### 原则 5：叙事逻辑必须通顺
不能出现"盖了盖子又打开"这种违反使用顺序的画面。每个镜头的内容必须和上下文衔接，放在这个位置要合理。

### 原则 6：时长不锁死，根据产品步骤来定
总时长控制在 **10-18 秒**。步骤多的产品可以长一点，步骤少的短一点。不要为了凑 15 秒而重复镜头或加无意义的画面。

### 原则 7：开头不要太碎
前半段不要全是 1.5 秒的短切。开头几段应该从动作的**自然起点**开始（通常是素材的 0 秒），给足呼吸。可以适当拉长到 2-2.5 秒。从素材中间位置截取容易导致动作不连贯。

### 原则 8：注意用户的特殊要求
比如"搓手指放最后"、"某个画面不要用"、"开盖画面去掉"。这些细节决定了视频的专业感。

### 原则 9：CTA 段永远放最后
引导点击的手势/画面不能出现在中间。

### 原则 10：同一条素材中有重复镜头时，只取最后一个
用户拍摄习惯是同一个动作拍多遍，一镜到底不停机。前面的都是"练习"，最后一遍才是满意的效果。所以当场景检测发现一条素材内有多个相似镜头时，**丢弃前面的，只保留最后一个**。

### 原则 11：成品中绝不出现重复镜头
严格去重。无论是同一条素材的不同时间段看起来一样，还是不同成品之间撞镜头，都不允许。如果可用镜头不够，宁可减少成品数量，也不要用重复画面凑数。

### 原则 12：镜头构图要有变化
连续的镜头不能都是同一种构图（比如连续三个都是"手拿产品正面"）。要在远景/近景/特写/使用画面之间交替，让视觉有节奏。

## 质量自检清单

制定方案后，逐条检查：

- [ ] 视频是否回答了「长啥样」？
- [ ] 视频是否回答了「怎么用」？
- [ ] 视频是否回答了「有什么用」？
- [ ] 每个镜头是否都在推进故事？（有没有同一步骤占了两个位置）
- [ ] 叙事顺序是否通顺？（有没有前后矛盾）
- [ ] 有没有重复镜头？
- [ ] 连续镜头的构图是否有变化？
- [ ] 总时长是否在 10-18 秒区间？
